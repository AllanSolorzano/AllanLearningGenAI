                IP
                
        Intent Pipeline (when Intent plan is on)
        
                
        
                  1. preparse → agent_preparse
                  2. memory retrieval → FTS + optional hybrid RAG
                  3. intent discovery → layered L1–L5 LLM JSON
                  4. capability + policy → agent_resolve
                  5. planner → step DAG (plan_graph)
                  6. verifier → replan if failures
                  7. kb_retrieval → kb_connector.query_local_kb
                  8. plan_executor → MCP tool calls
                  9. composer → evidence-based reply
                  10. memory_writer → data/memory/*.md
                  11. trace writer → data/traces/turn_*.md
                  12. orchestration audit → SQLite agent_* + orchestration_*

        Short-term Memory
        
              
        
                Recent conversation context loaded per turn from `chat_messages` (bounded by `history_limit`, default 40).
              
        
              
        
                SQLite
                bounded context
              
        
            
        
            
        
              
        Episodic Memory
        
              
        
                When `ALLAN_AGENT_MEMORY_WRITE=1`, the agent summarizes durable lessons and stores them as markdown in `data/memory//`.
              
        
              
        
                markdown
                FTS-indexed
              
        
            
        
            
        
              
        Long-term Memory
        
              
        
                Durable memory lives in SQLite + FTS5 (`durable_memory_fts`) with optional hybrid RAG (BM25 + embeddings + RRF) over chunked memory.
              
        
              
        
                FTS5
                hybrid RAG
              
        
            
        
            
        
              
        Reflective Memory
        
              
        
                Turn-by-turn pipeline traces written to `data/traces//turn_*.md` when `ALLAN_AGENT_TRACE_MD=1`.
              
        
              
        
                trace markdown
                auditability
              
        
            
        
            
        
              
        Procedural Memory
        
              
        
                The orchestration pipeline itself is persisted: intents, plans, tasks, tool calls, and KB queries in `orchestration_*` tables.
              
        
              
        
                pipeline state
                evidence bundle
              
        
            
        
            
        
              
        Memory Governance
        
              
        
                Policies in `orchestration_policies` can block unsafe memory writes (e.g., secrets) and require approval for certain state changes.
              
        
              
        
                guardrails
                audit log
              
        
            
        
          
        
        
        

        
        
          
        
            
        
              
        Web
        
              
        Static UI: `static/chat.html` with live SQLite trace panel.
        
            
        
            
        
              
        Backend
        
              
        FastAPI service (`allan_ollama_mcp/app.py`) with session + agent endpoints.
        
            
        
            
        
              
        LLM
        
              
        Ollama HTTP API for chat, plus optional tool calling (tools-capable model).
        
            
        
            
        
              
        Storage
        
              
        
                SQLite database (`data/app.db`) with FTS5 for durable memory.
              
        
            
        
            
        
              
        RAG (optional)
        
              
        
                `ALLAN_RAG_HYBRID=1` enables chunked hybrid retrieval using `sentence-transformers` and local BM25, fused with RRF.
              
        
            
        
            
        
              
        MCP tools
        
              
        
                MCP stdio servers configured via `mcp_remote_servers.json` and exposed through the Ollama tool registry.
              
        
            
        
          
        
        
        

      
        
    
        
  
        

  
        
    
        
      
        
        
        
          
        MCP servers (stdio)
        
          
        Same format as Cursor / VS Code mcp.servers: command, args (JSON array), optional cwd and env. Tools load per chat when MCP tools is enabled.
        
        
        
        
      
        
      
        
        
        Configured servers
        
        
        Disable a server to stop spawning its process. Delete removes it from mcp_remote_servers.json and clears saved tool toggles.
        
        
        

        
        
          
        Tool discovery (per MCP server)
        
          
        Saved tools load from SQLite. Discover tools connects to the MCP, refreshes the catalog, and updates descriptions. Disabled tools are not sent to the model on the next message.
        
          
        
            
            
          
        
          
        
            
        Select a server and click Discover tools.
        
          
        
        
        

        
        
          Advanced: edit JSON file
          
          
        
            
            
          
        
        
        

        
        
          
        
        
      
        
    
        
  
        

  
        
    
        
      
        
        
        
          
        Settings
        
          
        Stored in SQLite (app_settings). Ollama host is applied on save.
        
        
        
        
      
        
      
        
        
        
          Only Ollama (local) is fully wired in allan_project. Other providers are stored for future use.
